> [!NOTE] Key Message
> STROBE Statement は観察研究（cohort / case-control / cross-sectional の3デザイン）の報告品質を担保するための **22項目チェックリスト**。うち **18項目が3デザイン共通**、**4項目（item 6, 12, 14, 15）がデザイン固有**。項目はtitle/abstract（item 1）、introduction（2–3）、methods（4–12）、results（13–17）、discussion（18–21）、other information（funding, item 22）に対応。2004年9月にBristolで2日間workshop（参加者23名）を開催し、反復的なコンサルテーションを経て策定。**「報告の透明性」のためのガイドであり、研究の質を評価するツールでも、デザイン・実施の処方箋でもない**と明言。SAKURA-AVRは投稿時にSTROBE準拠が必須であり、本チェックリストの各項目（特に交絡調整・バイアス・脱落・一般化可能性）に対応する形で原稿構成を組む。

## 1. 背景・目的

- 生物医学研究の多くは観察研究であり、その報告はしばしば不十分で、研究の強み・弱み・一般化可能性の評価を妨げている。
- ランダム化試験では答えられない疑問（稀・遅発性の有害作用、日常診療での実際の達成度）に観察研究が有用。
- 既存の報告不備のエビデンス例として本文が引用するもの:
  - 脳卒中研究の縦断研究調査で **49本中17本（35%）が適格基準を明記していなかった**（Tooth 2005）。
  - 精神科のcase-control研究で、症例・対照の同定法を説明していた報告はごくわずか（Lee 2007）。
  - 疫学研究で潜在的交絡因子の選択根拠がしばしば未報告（Pocock 2004）。
- RCT報告のCONSORT（1996策定・5年後改訂）が報告品質を改善した実績に倣い、観察研究版として **STROBE Initiative**（2004年設立）を立ち上げた。
- スコープは解析的観察研究で最も広く使われる **cohort / case-control / cross-sectional の3デザイン**に限定（case-crossover・ecological等は対象外、将来拡張に委ねる）。

## 2. 中身（チェックリストの構造と項目＝SAKURAで引用する具体値）

### 2.1 全体構造
- 総項目数: **22項目**。
- **18項目が3デザイン共通**、**4項目がデザイン固有**: item **6**（participants）、**12**（statistical methods）、**14**（descriptive data）、**15**（outcome data）。一部はサブ項目でデザイン別バージョンを持つ。
- アスタリスク付き項目（8, 13, 14, 15）は、case-control研究では症例・対照ごとに、cohort/cross-sectionalでは曝露群・非曝露群ごとに **別々に情報を記載**すべき。
- 詳細な根拠・方法論的背景・良い報告の実例は別冊 **Explanation and Elaboration（E&E）論文**（PLoS Med 4:e297）に収載。チェックリストはE&Eと併用が推奨される。

### 2.2 セクション別の22項目（SAKURA原稿で対応づける具体項目）

| セクション | item | 推奨内容（要点） |
|---|---|---|
| Title/Abstract | 1 | (a) タイトルか抄録で研究デザインを一般的用語で明示 (b) 抄録に「何をして何が分かったか」のバランスの取れた要約 |
| Introduction | 2 | 科学的背景とrationale |
| | 3 | 具体的目的（**事前規定された仮説を含む**） |
| Methods | 4 | 研究デザインの主要素を論文冒頭で提示 |
| | 5 | setting・場所・関連日付（**recruitment / exposure / follow-up / データ収集の期間**） |
| | 6 | （cohort）適格基準、参加者選択の源と方法、follow-up法。マッチド研究では**マッチ基準と曝露/非曝露の数** |
| | 7 | アウトカム・曝露・予測因子・**潜在的交絡因子**・効果修飾因子をすべて明確に定義。診断基準を提示 |
| | 8* | 各変数のデータ源と測定法の詳細。群が複数なら測定法の比較可能性を記述 |
| | 9 | **バイアス源への対処の記述** |
| | 10 | study sizeの決め方 |
| | 11 | 量的変数の扱い（カテゴリ化した場合はそのグルーピングと理由） |
| | 12 | (a) **交絡を制御する手法を含む全統計手法** (b) サブグループ・交互作用の検討法 (c) **欠測データの扱い** (d) cohortでは**脱落（loss to follow-up）の扱い** (e) **感度分析** |

| Other | 22 | **資金源と資金提供者の役割**（該当すれば元研究のものも） |

### 2.3 開発プロセス（数値）
- 2004年9月、Bristol（UK）で **2日間workshop**を開催、**23名**が参加（Annals of Internal Medicine、BMJ、Bulletin of WHO、Int J Epidemiology、JAMA、Preventive Medicine、Lancetの編集者を含む）。
- 出席できない**10名**から書面寄稿。3つのworking groupでデザイン別に項目を抽出。
- workshop後、coordinating group（本論文著者）が **8回**会合し、Web上で**3回の改訂**と**2回のコメント要約**を公開。**30名超**からフィードバックを得た。

## 3. 本研究での使い方（SAKURA-AVRでの採用と§対応）

SAKURAは日韓多施設**後ろ向きコホート**（PSM/IPTW）であり、投稿時にSTROBE準拠が必須。各項目を以下のように原稿§へマッピングする。

- **item 1（デザイン明示）**: タイトル・抄録に「retrospective multicenter cohort, propensity-matched」を明記。
- **item 5（期間）**: 各施設の登録期間・追跡期間を Methods に明記。
- **item 6/7（適格基準・変数定義）**: 「measured annulus diameter ≤23mm（primary stratum ≤21mm）」「small-body Asian AS」の **適格基準**と、Arm1（no-AAE SAVR）/ Arm2（AAE-SAVR: Nicks/Manouguian/Konno/Y-incision）/ reference arm（self-expanding TAVI）の**群定義**を§対象・方法に記載。
- **item 7/8/12（アウトカム・測定・統計）**: co-primary endpoint（**measured iEOA ≤0.65 cm²/m² によるsevere PPM** と1年平均圧較差）、VARC-3エンドポイント定義、PSM/IPTW・多重代入による交絡調整を §統計解析に記載。item 16の **95% CI付き推定値**もここに対応。
- **item 9（バイアス）**: 適応交絡・施設選択バイアスへの対処（PSM/IPTW、E-value による未測定交絡感度）を記載。
- **item 12c/13/14b（欠測・フロー・欠測者数）**: 多重代入の方法、各段階の患者数フロー図、変数別欠測数を §結果に記載。
- **item 12e/17（感度分析）**: E-value、IPTW vs PSM の感度分析、prespecified safety（post-AAE mitral regurgitation）の追加解析を記載。
- **item 19（限界）**: 後ろ向きデザイン・残存交絡のバイアスの方向と大きさを議論。
- **item 21（一般化可能性）**: Asian small-body集団に限定した外的妥当性（非アジア集団・大体格への外挿の限界）を議論。
- **item 22（funding）**: 資金源を明記。

→ 実務上は **STROBE cohort版チェックリスト**（Web版）を投稿時の補足資料として添付し、各項目の記載ページ番号を埋める。

## 4. 注意点

- STROBEは **報告の透明性を担保するガイドであって、研究の質を評価するツールではない**（質評価尺度は別途存在、Sanderson 2007のsystematic reviewが対象）。SAKURAの質はSTROBE準拠の有無では判定されない。
- **デザイン・実施の処方箋ではない**。チェックリストに沿って「STROBE化（"STROBEd"）」して文体・用語を規制することは意図されていない。叙述的記述（症例例示など）の併用を推奨。
- 各項目は **論文のどこかで十分な詳細と明確さをもって扱われていればよく**、順序・体裁は著者・誌・分野の慣習に委ねられる（例: item 22 fundingは付録でも可、resultsの複数項目を1つの表でまとめても可）。
- 対象は3デザインに限定。SAKURAはcohortに該当するためデザイン固有項目（6/12/14/15）はcohort版を採用。case-control固有・cross-sectional固有のバージョンは無関係。
- STROBEは「進化する文書」であり将来改訂・拡張があり得る（STREGA等の拡張が存在）。投稿先誌の最新版チェックリスト要件を確認すること。
- 開発グループは地理的にヨーロッパ・北米に偏っており、研究関心・分野の代表性にも限界がある旨を著者自身が limitation として明記している。
